## Documentation Naming Convention

For dated documentation (implementation guides, migration notes, etc.), use the format:
```
DOCUMENT_NAME_YYYY-MM-DD.md
```

Examples:
- `SEO_FILES_IMPLEMENTATION_2026-02-01.md`
- `DATABASE_MIGRATION_2026-03-15.md`
- `API_REFACTOR_2026-04-20.md`

For general documentation (guides, architecture docs, etc.), use descriptive names without dates:
- `API_GUIDE.md`
- `ARCHITECTURE.md`
- `DEPLOYMENT.md`
